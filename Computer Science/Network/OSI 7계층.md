# OSI 7계층 정리

## 0. 개요 
### * OSI 7계층이란?
오픈 시스템 상호 연결, Open Systems Interconnections(OSI) <br>
국제표준화기구(ISO)가 정의한 네트워크 통신 모델 <br>
OSI 모델은 네트워크 통신 기능을 7개의 계층으로 나누는 개념적 프레임 워크<br>

**통신이 일어나는 과정을 단계별로 알 수 있고, 특정한 곳에 이상이 생기면 해당 단계만 수정할 수 있기 때문**

---

### * OSI 모델이 중요한 이유
OSI 모델은 소프트웨어 및 하드웨어 구성 요소 전반에 걸쳐 모든 유형의 네트워크 통신을 캡슐화한다.<br>

**1. 복잡한 시스템에 대한 이해 공유** <br>
* OSI 모델을 사용하여 복잡한 네트워크 시스템 아키텍처를 구성하고 모델링할 수 있음
* 주요 기능에따라 각 시스템 구성 요소의 운영 계층을 분리할 수 있음
* 추상화를 통해 시스템을 더 작고 관리 가능한 부분으로 분해 -> 시스템을 전체적으로 쉽게 개념화 할 수 있음
  
**2. 더 빠른 연구 및 개발** <br>
* 엔지니어가 작업을 더 잘 이해할 수 있음
* 새로운 시스템을 만들 때 어떤 기술 꼐층을 대상으로 개발하는지 알 수 있음

**3. 유연한 표준화** <br>
* 낮은 수준의 네트워킹 및 프로토콜을 추상화하여 시스템 설계 및 개발을 단순화 함
---

## 1. 계층별 주요 기능

| 계층 | 계층명 | 주요 기능 및 역할 |
|:--:|:--:|:--|
| **7** | **응용 계층 (Application)** | • **사용자 계층**<br>• 사용자와 직접 상호작용<br>• 응용 프로그램 간 데이터 교환<br>• 네트워크 서비스 요청 및 응답 처리<br>•사용자 인터페이스, 전자우편, 데이터베이스 관리 등의 서비스 제 |
| **6** | **표현 계층 (Presentation)** | •**표현 계층**<br>• 데이터의 형식 변환, 암호화, 압축 수행<br>• 서로 다른 시스템 간의 데이터 형식 일치 |
| **5** | **세션 계층 (Session)** | •**세션 관리 및 포트 전송**<br>• 통신 세션의 설정, 유지, 종료 관리<br>•데이터가 통신하기 위한 논리적 연결 담당<br>•TCP/IP 세션을 만들고 없애는 책임<br>• 송수신 간 동기화 기능 제공 |
| **4** | **전송 계층 (Transport)** | •**종단 간 연결 관리**<br>• 신뢰성 있는 데이터 전송 제공<br>•TCP와 UDP 프로토콜을 통해 통신 활성화<br>•포트를 열어두고 프로그램들이 전송할 수 있도록 제공<br>• 패킷 분할·재조립, 오류 검출, 흐름 제어 수행 |
| **3** | **네트워크 계층 (Network)** | •**패킷 단위데이터 전송**<br> • 경로 설정 및 라우팅 기능<br>• **논리적 주소(IP)** 관리 및 패킷 전달 |
| **2** | **데이터링크 계층 (Data Link)** | •**프레임 단위 전송**<br>• **물리적 주소(MAC) 기반** 데이터 프레임 전송<br>• 오류 검출 및 재전송 기능 |
| **1** | **물리 계층 (Physical)** | •**물리적 전송 구조**<br>• 비트 단위의 전송 담당<br>• 전기적 신호, 커넥터, 전송 매체 규격 정의 |

## 2. 계층별 대표 프로토콜

| 계층 | 대표 프로토콜 |
|:--:|:--:|

| **6 표현 계층** | **SSL/TLS**, JPEG, MPEG, IMAP, ASCII, SSH |

| **4 전송 계층** | **TCP**, **UDP** |
| **3 네트워크 계층** | **IP**, **ICMP**, IPSec, IGMP, **ARP**, **RARP**, **OSPF**, **RIP** |
| **2 데이터링크 계층** | **Ethernet**, PPP, HDLC|
| **1 물리 계층** | IEEE 802.3, RS-232 |

## 💡 암기 팁

> “**물-데-네-전-세-표-응**”  
> 물리 → 데이터링크 → 네트워크 → 전송 → 세션 → 표현 → 응용  
> 아래로 갈수록 하드웨어 중심, 위로 갈수록 사용자 중심

## (추가)OSI 7계층 ↔ TCP/IP 4계층 매핑 정리

### 1️. OSI 7계층 vs TCP/IP 4계층 비교

|     구분     | OSI 7계층         | TCP/IP 4계층                               | 주요 역할                                           |
| :--------: | :-------------- | :--------------------------------------- | :---------------------------------------------- |
| **7~5 계층** | 응용 / 표현 / 세션 계층 | **응용 계층 (Application Layer)**            | 사용자가 직접 이용하는 응용 서비스 제공 (HTTP, FTP, SMTP, DNS 등) |
|  **4 계층**  | 전송 계층           | **전송 계층 (Transport Layer)**              | TCP/UDP를 이용한 종단 간 신뢰성 있는 데이터 전송                 |
|  **3 계층**  | 네트워크 계층         | **인터넷 계층 (Internet Layer)**              | IP를 이용해 경로를 설정하고 패킷을 목적지까지 전달                   |
| **2~1 계층** | 데이터링크 / 물리 계층   | **네트워크 접근 계층 (Network Interface Layer)** | 실제 물리 매체를 통해 데이터를 전송 (MAC, Ethernet 등)          |

---

### 2️. TCP/IP로 나누는 이유

| 구분           | OSI 모델                    | TCP/IP 모델                  |
| :----------- | :------------------------ | :------------------------- |
| **성격**       | 이론적 표준 모델 (통신 과정의 개념적 정의) | 실제 인터넷 통신 프로토콜 기반 구현 모델    |
| **계층 수**     | 7계층                       | 4계층                        |
| **목적**       | 네트워크 설계와 학습을 위한 참고 모델     | 인터넷 전송을 위한 실질적 프로토콜 체계     |
| **주요 사용 예시** | 네트워크 이론, 시험(정보처리기사 등)     | 실제 네트워크 통신 (웹, 이메일, 라우팅 등) |

* **OSI 모델** → “교과서 모델” (각 역할을 세분화해 **이론적으로 구분**)
* **TCP/IP 모델** → “실무 모델” (실제 인터넷 통신이 **이 구조로 동작**)
* 즉, **TCP/IP는 OSI 7계층을 단순화**해서 실제 구현에 맞게 4계층으로 통합한 형태.
